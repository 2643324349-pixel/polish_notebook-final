# 核心交互逻辑

> **实现平台**：Flutter（跨平台：Web / iOS / Android）
> **组件映射**：本文档中的"页面"对应 Flutter 的 `Screen` 或 `View`，"模态框"对应 `Dialog` 或 `BottomSheet`，"列表"对应 `ListView` 或 `GridView`
## 1. 笔记本管理

| 操作 | 用户触发 | 系统响应 | 异常处理 |
|------|---------|---------|---------|
| **新建笔记本** | 点击「+ New Notebook」 | 弹出模态框→输入名称→确认→列表顶部新增→自动跳转至笔记本内（空状态，无页） | 名称为空禁用确认；重名提示"已存在" |
| **删除笔记本** | 点击笔记本卡片上的「⋮」→「Delete」 | 弹出二次确认→确认后删除所有关联Sheet/Row→列表刷新 | 删除中若失败，Toast提示并回滚 |
| **重命名笔记本** | 双击笔记本名称 | 变为可编辑输入框→失焦或回车保存 | 空名称恢复为"Untitled" |
| **笔记本排序** | 长按/拖拽笔记本卡片 | 拖拽排序（同页/行/列排序规则） | 同下方拖拽规范 |

## 4. 拖拽排序规范（列/行/页/笔记本通用）

### 触发方式
- **Web**：鼠标悬停显示拖拽手柄（⋮⋮）→ 按住拖动
- **移动端**：长按手柄区域 0.3s → 拖动

### 视觉反馈
- **拾取**：元素浮起（阴影+透明度80%），原位灰色虚线占位符
- **移动**：其他元素平滑让位（200ms过渡动画）
- **释放**：元素落入新位置，阴影消失

### 数据更新
- **乐观更新**：前端立即更新顺序
- **异步保存**：防抖500ms后将新ID顺序数组发送后端

### 异常处理
- 拖出有效区域 / 按 Esc 取消：弹回原位（Spring动画）
- 跨层级拖拽（如把行拖到列区域）：松手时无效并弹回
- **锁定项**：首列（主格列）禁止移动
- **模式冲突**：批量选择模式下全局禁用拖拽排序
- **大数据量**：行/列数 > 100 时，关闭平滑让位动画，仅显示蓝色插入指示线
- **同步失败**：Toast提示"排序同步失败"，前端保留当前新顺序，下次操作静默重试

## 5. 一键补全（✨ Generate）

| 步骤 | 说明 |
|------|------|
| **1. 形态学分析** | 调用规则引擎（Morfeusz 抽象层），识别：原形(Lemma)、词性(POS)、语法特征(性/数/格) |
| **2. 判断是否需要多性别** | 如果词性为**形容词、指示代词、数字（2，3，4）数词** → 需要生成3种性 |
| **3. 生成多性别值** | 引擎返回：`{ m: "dobry", f: "dobra", n: "dobre" }` |
| **4. 智能填充** | 将三种性分别填入同一单元格的 `gender_values.m/f/n` |

### UI 展示

三种性在**同一行内垂直排列**，共用"翻译"列：

| 翻译 | 单词(主格) | 属格 |
|-----------|------|------|
| 好的 | **dobry (m)** | **dobrego (m)** |
| ↑ 合并单元格 | **dobra (f)** | **dobrej (f)** |
| 跨行居中）| **dobre (n)** | **dobrego (n)** |

### 异常处理
- **未收录/非法词汇**：Toast提示"词典未收录该词或拼写错误"，不执行填充
- **多义词/多性歧义**：弹出底部半屏弹窗（Action Sheet）列出所有可能选项供用户选择（如"1. 名词-阴性 / 2. 形容词-阴性"）
- **性能异常（>500ms）**：按钮Loading超过1s显示微动效，超过3s Toast提示"处理较慢，请稍候"
- **目标单元格隐藏**：仅更新底层数据，UI保持隐藏状态

## 7. 批量选择与显隐切换

### 进入/退出选择模式
- **进入**：长按单元格内容区域 0.5s / 点击右上角「选择」按钮
- **退出**：点击「完成」/ 点击空白区域 / 按返回键（移动端）

### 选择操作
| 操作 | Web 桌面端 | 移动端 |
|------|-----------|--------|
| **单选** | 点击单元格勾选框 | 点击单元格勾选框 |
| **全选/取消全选** | 顶部「全选」复选框 | 顶部「全选」复选框 |
| **连续多选** | Shift + 点击首尾行 | 滑动勾选框（iOS照片同款手势） |
| **离散多选** | Ctrl/Cmd + 点击 | 逐个点击勾选框 |
| **键盘快捷键** | Ctrl+A 全选，Esc 退出 | — |

### 底部操作栏
- 显示 **"已选 N 项"**
- **「🙈 隐藏」**：将所有已选单元格设为隐藏态，隐藏单元格内的**所有性别值**（m/f/n全部隐藏）
- **「👁️ 显示」**：将所有已选单元格恢复原文（仅对已隐藏的生效），恢复单元格内的**所有性别值**

### 约束
- 选择模式下**禁止编辑、AI生成、拖拽排序**等所有非选择操作
- 隐藏/显示操作完成后，**自动退出选择模式**并清空选中状态
- 若选中行中包含备注列，备注列**不参与**显隐切换（保持始终可见）

## 8. 导出（PNG / Excel）

工具栏按钮顺序：**搜索 → 导出 → 选择**（位于 Sheet 页顶部工具栏）。

| 操作 | 用户触发 | 系统响应 | 异常处理 |
|------|---------|---------|---------|
| **打开导出菜单** | 点击「Export」 | 弹出菜单：「Export as PNG」「Export as Excel」 | 批量选择模式 / 同步中禁用 |
| **导出 Excel** | 菜单 → Export as Excel | 仅导出**当前可见列** × 全部行 → 生成 `.xlsx` → 下载或系统分享 | 超过 5000 行显示 loading toast；失败 Toast |
| **导出 PNG（短表）** | 菜单 → Export as PNG | 表格高度 ≤ 可视区域 → 直接截取可视区域 → 单张 PNG 下载 | 失败 Toast + 长按保存提示 |
| **导出 PNG（长表）** | 菜单 → Export as PNG | 弹出范围选择：「可视区域」/「全部内容（多张 PNG）」 | 取消则关闭对话框 |

### Excel 规则

- **列范围**：与界面一致，仅 `visibleColumns`（隐藏列不导出）
- **行范围**：全部数据行（含表头行）
- **列宽**：按可见列单元格内容动态计算（CJK 字符按 2 宽度计，上限 60）
- **格式**：数字 `#,##0.00`，文本 `@`
- **文件名**：`vocab-export-{YYYYMMDDHHmmss}.xlsx`
- **分享**：优先 Web Share API；不支持时降级为浏览器下载

### PNG 规则

- **渲染**：`dom-to-image-more`，等待 `document.fonts.ready`，`devicePixelRatio × 2`
- **字体**：Inter / Noto Sans / Noto Sans Cyrillic
- **可视区域**：1 张，`vocab-export-{YYYYMMDDHHmmss}.png`（无页码编号）
- **全部内容**：按可视高度分页，逐张下载 `vocab-export-1.png`、`vocab-export-2.png` …（不使用 zip 打包）
- **移动端**：优先系统分享面板；降级时长按保存提示

### 单元格导出文本

- 多行/多性别值用 ` / ` 连接
- 隐藏单元格导出为 `•••`

## 10. 搜索单词（查找/高亮）

| 操作 | 用户触发 | 系统响应 |
|------|---------|---------|
| **打开搜索** | 点击工具栏「🔍」/ 按 `Ctrl+F` | 展开搜索框，自动聚焦 |
| **输入关键词** | 在搜索框中输入 | 实时高亮匹配单元格，显示「找到 N 个匹配项」 |
| **下一个** | 点击「↓」/ 按 `Enter` | 跳转到下一个匹配项，滚动到可视区域 |
| **上一个** | 点击「↑」/ 按 `Shift+Enter` | 跳转到上一个匹配项 |
| **关闭搜索** | 点击「×」/ 按 `Esc` | 关闭搜索框，清除所有高亮 |

**搜索规则：**
- 范围：当前表格所有可见列 × 所有行
- 匹配：不区分大小写，部分匹配
- 隐藏单元格：用原始 value 匹配

**视觉反馈：**
- 所有匹配项：浅黄背景（`bg-amber-100`）
- 当前激活项：`ring-2 ring-amber-500`
- 搜索框显示「找到 N 个匹配项」
